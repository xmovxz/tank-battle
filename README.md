# T1-T4 层级会议问题管理系统

> 面向制造型企业的现场 / 生产异常问题闭环管理工具，支持 T1→T4 层级逐级升级、本级闭环处置、全生命周期追溯。

---

## 层级体系说明

| 层级 | 名称 | 角色 | 职责 |
|------|------|------|------|
| **T1** | 班组 | 一线班组长 / 操作员 | 日常巡检、问题发现与上报、简单问题本级处置 |
| **T1 PLUS** | 车间主任 | 车间负责人 | 跨班组协调、设备维修调度、工艺调整 |
| **T2** | 厂部经理 | 部门经理 | 资源调配、跨部门协调、采购申请审批 |
| **T3** | 分管副总 | 副总裁 | 重大资源决策、供应商管理、体系优化 |
| **T4** | 总经理 | 最高管理层 | 重大安全 / 合规 / 预算决策 |

### 升级规则

- 问题可以从 **T1 → T1 PLUS → T2 → T3 → T4** 逐级升级，**不可越级**
- 当本级无法独立处置问题时，在层级会议中决策是否向上一级升级
- 升级时必须填写升级说明和操作人
- 任何层级均可进行**本级闭环处置**（在本级解决问题并关闭）
- T4 为最高层级，无法继续升级；若 T4 仍无法处置，需转专项项目

## 功能特性

### 1. 问题录入
- 支持现场异常、生产异常等各类问题录入
- 必填字段：标题、类别、严重度、来源、报告人、描述
- 自动生成唯一编号（格式：`ISS-YYYYMMDD-NNN`）

### 2. 逐级升级
- 严格按 T1→T1P→T2→T3→T4 链路升级
- 每次升级需填写升级说明，说明原因及需上级协调事项
- 升级操作完整记录至生命周期追溯链

### 3. 本级闭环
- 任何层级均可关闭问题
- 闭环时需填写闭环原因及处理措施
- 记录闭环层级、操作人及时间

### 4. 全生命周期追溯
- 完整记录问题从创建到闭环的全过程
- 每次操作（创建、升级、闭环、重新打开、评论、编辑）均带时间戳和操作人
- 可视化时间轴展示

### 5. 数据管理
- 所有数据存储在浏览器 localStorage 中
- 支持 JSON 格式导出 / 导入备份
- 支持清除重置

## 使用方法

### 打开系统
直接用浏览器打开 `index.html` 文件即可，无需安装任何软件或启动服务器。

### 浏览器兼容性
- ✅ Google Chrome（推荐）
- ✅ Microsoft Edge（推荐）
- ✅ Firefox
- ✅ Safari
- ⚠️ 需要支持 ES6+ 的现代浏览器

### 各视图说明

| 视图 | 功能 |
|------|------|
| **仪表盘** | 查看问题总数、未闭环数、已升级数、紧急问题数，各级别分布图及最近动态 |
| **问题列表** | 按类别/严重度/层级/状态筛选，支持关键词搜索，点击行进入详情 |
| **录入问题** | 新建问题，选择初始层级（默认 T1） |
| **问题详情** | 查看完整信息、执行升级/闭环/重新打开操作、添加评论、查看生命周期时间轴 |

### 典型操作流程

```
1. 巡检发现问题 → 录入问题（T1 班组）
2. T1 会议讨论 → 班组无法独立处理 → 升级至 T1 PLUS
3. 车间主任分析 → 需厂部协调资源 → 升级至 T2
4. 厂部经理处理 → 调配资源解决问题 → 本级闭环（T2）
5. 全流程可追溯，每个操作节点清晰记录
```

### 数据备份

- 点击任意页面的「导入/导出」按钮
- 导出：下载 JSON 备份文件
- 导入：从备份文件恢复数据（与现有数据合并）
- 建议定期导出备份

## 技术实现

- **纯前端**：HTML + CSS + JavaScript，无框架依赖
- **数据存储**：浏览器 localStorage
- **路由**：Hash-based SPA（`#dashboard`、`#list`、`#create`、`#detail/{id}`）
- **响应式**：适配桌面端、平板和手机
- **离线可用**：完全离线运行，无需网络连接
